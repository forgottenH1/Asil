# نظام إدارة وتطوير مشروع "المنصة الإسلامية المتكاملة"

## 1. رؤية المشروع
إنشاء منصة إسلامية متطورة تعتمد على تقنيات Serverless، تقدم تجربة مستخدم فريدة لمحاكاة المواريث، الاستماع للقرآن، ومتابعة التقويم الهجري ومواقيت الصلاة، مع تركيز عالٍ على الأداء وجماليات التصميم (Modern UI/UX).

## 2. المكدس التقني (Tech Stack)
- **Frontend:** React.js (Vite) - لضمان أقصى سرعة في التطوير والتشغيل.
- **Styling:** Tailwind CSS + Framer Motion - لتحقيق مظهر عصري (Glassmorphism) يشبه منصة "درر".
- **Deployment:** Cloudflare Pages (Serverless) - لضمان استجابة سريعة وتوسع تلقائي.
- **APIs:** - Quran: Mp3Quran API.
  - Prayer Times: Aladhan API (تحديث تلقائي ودقيق).
  - Logic: JavaScript Engine لعمليات الإرث المعقدة (Client-side processing).

## 3. بنية الصفحات والمسارات (Architecture)

### أ. تطبيق حساب المواريث (Inheritance Simulator)
- **الوظيفة:** محرك منطقي لحل المسائل المعقدة (أصحاب الفروض، العصبات، الحجب، الرد، والعول).
- **الميزة الإضافية:** عرض "الدليل الشرعي" (آية أو حديث) بجانب كل نتيجة تظهر للمستخدم.
- **التصميم:** بطاقات تفاعلية (Interactive Cards) تظهر خطوات الحل.

### ب. قسم القرآن الكريم (Audio Hub)
- **المصدر:** التكامل مع Mp3Quran API.
- **المشغل:** Modern Audio Player مخصص يدعم (الاستماع في الخلفية، التحميل المباشر، اختيار القراء).
- **التصميم:** شبكة (Grid) تحتوي على سور القرآن مع خاصية البحث السريع.

### ج. التقويم الهجري والميلادي
- **الوظيفة:** عرض اليومية الهجرية مع ربطها بالأحداث الإسلامية.
- **التقنية:** استخدام مكتبة `hijri-converter` لضمان الدقة العالية.

### د. مواقيت الصلاة
- **الوظيفة:** تحديد الموقع التلقائي (Geolocation) وعرض المواقيت لكل مدن العالم.
- **الميزة:** عداد تنازلي للصلاة القادمة وتنبيهات بصرية.

## 4. تحسين محركات البحث (SEO Strategy)
- استخدام **React Helmet** لكل صفحة بشكل مستقل.
- كلمات مفتاحية استراتيجية في الـ Metadata (مثل: حساب المواريث أونلاين، استماع القرآن الكريم MP3، مواقيت الصلاة بدقة).
- إنشاء ملف `sitemap.xml` و `robots.txt` برمجياً عند البناء.

## 5. خوارزمية العمل (Development Workflow)

### المرحلة الأولى: الإعداد (Setup)
1. تهيئة مشروع Vite React.
2. تثبيت Tailwind CSS وتهيئة الخطوط (عربي/إنجليزي) لتشبه خطوط موقع "درر".
3. إعداد بنية المجلدات (Components, Hooks, Pages, Utils).

### المرحلة الثانية: تطوير الواجهات (UI Development)
1. بناء الـ Header: يحتوي على روابط السياسات (Privacy Policy) وخدمات الموقع.
2. بناء الـ Footer: يحتوي على حقل بحث ذكي، حقوق الملكية، وروابط التواصل (GitHub, LinkedIn).
3. تطبيق نظام الـ Responsive لضمان عمل الموقع على الهواتف بنفس كفاءة الحاسوب.

### المرحلة الثالثة: الربط البرمجي (Backend-less Logic)
1. بناء محرك المواريث كـ Pure JavaScript Functions لضمان السرعة والخصوصية (Client-side).
2. ربط الـ APIs الخارجية باستخدام `Axios` مع معالجة الأخطاء.

### المرحلة الرابعة: النشر (Deployment)
1. ربط مستودع GitHub بـ Cloudflare Pages.
2. إعداد الـ Environment Variables (إن وجدت).
3. تفعيل الـ SSL و Brotli Compression لضغط الملفات.

## 6. إضافات مقترحة للكمال البرمجي
- **Dark Mode:** زر تبديل بين الوضع الليلي والنهاري.
- **PWA:** تحويل الموقع إلى تطبيق ويب تقدمي يمكن تثبيته على الهاتف.
- **Copy Results:** زر لنسخ نتيجة مسألة الإرث مع دليلها الشرعي لمشاركتها عبر الواتساب.
